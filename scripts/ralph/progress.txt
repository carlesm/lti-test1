## Codebase Patterns
- Use `uv run python manage.py ...` instead of `python manage.py ...` (uv-managed environment)
- Django project name: `lti_project_selection`; main app: `projects`
- Settings reads SECRET_KEY, DATABASE_URL, ALLOWED_HOSTS from env; SQLite fallback for dev
- WhiteNoise is configured for static files (both INSTALLED_APPS and MIDDLEWARE)
- Templates live in `BASE_DIR / 'templates'`; base.html uses Bootstrap 5.3.3 CDN
- No type checker (mypy/ty) installed; typecheck = `uv run python -m py_compile <files>` + `manage.py check`
- django-lti package name is `lti_tool` (not `django_lti`); INSTALLED_APPS entry is `"lti_tool"`
- LTI "Platform" is called `LtiRegistration` in lti_tool (fields: issuer, client_id, auth_url, token_url, keyset_url)
- lti_tool uses `DjangoCacheDataStorage` (pylti1p3) → requires CACHES with DatabaseCache; run `createcachetable` after migrate
- LTI URL pattern: login at `/lti/login/`, launch at `/lti/launch/`, JWKS at `/.well-known/jwks.json`
- Launch view lives in `projects/lti_views.py`; extends `LtiLaunchBaseView`, implements `handle_resource_launch`
