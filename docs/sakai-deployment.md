# Deploying the LTI Project Selection Tool in Sakai

This guide walks through registering and deploying the LTI Project Selection tool in a Sakai LMS instance (Sakai 21+, which includes full LTI 1.3 support).

---

## Prerequisites

- The application is deployed and accessible over HTTPS (e.g. `https://lti-projects.your-institution.edu`)
- You have **Sakai administrator** access to the Administration Workspace
- You have **Django admin** access to the LTI Project Selection application

---

## Part 1: Register the Tool in Sakai (Admin)

### Step 1: Open the LTI Admin panel

1. Log in to Sakai as an administrator
2. Navigate to **Administration Workspace** (the wrench icon in the top navigation)
3. In the left sidebar, click **External Tools**

### Step 2: Create a new LTI 1.3 tool

1. Click **Install LTI 1.x Tool**
2. Fill in the tool configuration:

   | Field | Value |
   |-------|-------|
   | **Tool Title** | LTI Project Selection |
   | **Launch URL** | `https://lti-projects.your-institution.edu/lti/launch/` |
   | **LTI Version** | LTI 1.3 |

3. Expand the **LTI 1.3** section and note the values Sakai provides — you will need these in Part 2:
   - **Issuer** (Sakai's base URL, e.g. `https://sakai.your-institution.edu`)
   - **Client ID** (generated by Sakai for this tool registration)
   - **Platform OIDC Auth URL**
   - **Platform OAuth2 Well Known / KeySet URL**
   - **Platform OAuth2 Bearer Token Retrieval URL**

4. Fill in your tool's LTI 1.3 endpoints:

   | Field | Value |
   |-------|-------|
   | **Tool Keyset URL** | `https://lti-projects.your-institution.edu/.well-known/jwks.json` |
   | **Tool OpenID Connect/Initialization Endpoint** | `https://lti-projects.your-institution.edu/lti/login/` |
   | **Tool Redirect URL** | `https://lti-projects.your-institution.edu/lti/launch/` |

5. Under **Services**, enable:
   - **Assignments and Grades** — required for grade passback
   - **Names and Role Provisioning** — optional, useful for displaying student names

6. Click **Save**

---

## Part 2: Register Sakai as a Platform in Django Admin

1. Log in to the Django admin at `https://lti-projects.your-institution.edu/admin/`
2. Navigate to **Lti → Lti 1.3 Platforms → Add**
3. Fill in the values collected from Sakai in Step 2 above:

   | Django Admin Field | Sakai Value |
   |-------------------|-------------|
   | **Issuer** | Sakai's base URL (e.g. `https://sakai.your-institution.edu`) |
   | **Client id** | Client ID generated by Sakai |
   | **Auth login url** | Platform OIDC Auth URL |
   | **Auth token url** | Platform OAuth2 Bearer Token Retrieval URL |
   | **Key set url** | Platform OAuth2 Well Known / KeySet URL |

4. Click **Save**

---

## Part 3: Add the Tool to a Course (Professor)

Once the tool is registered globally, professors can add it to their individual courses.

### Option A: Via Site Info > External Tools

1. In your course site, click **Site Info** in the left menu
2. Click **External Tools**
3. Click **Select LTI Tool**
4. Select **LTI Project Selection** from the list
5. Click **Enable** and then **Continue**
6. The tool will appear as a link in the course left navigation

### Option B: Via Lessons

1. In your course site, open **Lessons**
2. Click **Add Content → Add External Tool**
3. Select **LTI Project Selection**
4. Save the page

### First launch

When a professor clicks the tool link for the first time, a Course record is automatically created in the application. The professor can then start creating projects immediately.

---

## Part 4: Verify the Integration

### Check the LTI launch works

1. As a professor, click the tool link in your course
2. You should see the **Professor Dashboard** with the course title and phase "Setup"
3. If you see a 403 or error page, check that:
   - The Platform record in Django admin matches the values from Sakai exactly (no trailing slashes unless Sakai includes them)
   - The tool's JWKS URL is reachable from the Sakai server
   - `ALLOWED_HOSTS` in `.env` includes the application's domain

### Check grade passback (AGS)

1. Complete the full workflow: create projects → open selection → students submit preferences → run assignment → publish results
2. After publishing, open the Sakai gradebook for the course
3. Students with an assigned project should show a score of **1.0**
4. Students without an assignment should have an **empty** grade column

### Troubleshooting

| Symptom | Likely cause |
|---------|-------------|
| `403 Forbidden` on launch | LTI session invalid; check issuer/client_id match exactly |
| `Invalid id_token` error | Clock skew between servers; ensure both servers use NTP |
| JWKS endpoint unreachable | Firewall blocking Sakai → app; check network/TLS |
| Grade passback silently skipped | `lineitem_url` not included in launch; ensure AGS service is enabled on the Sakai tool |
| Student sees professor view | LTI `roles` claim not sent by Sakai; check tool configuration sends role information |

## Sakai-Specific URL Reference

For a Sakai instance at `https://sakai.your-institution.edu`, the standard LTI 1.3 endpoint URLs are:

| Endpoint | URL |
|----------|-----|
| OIDC Auth URL | `https://sakai.your-institution.edu/imsblis/lti13/oidc_auth` |
| Token URL | `https://sakai.your-institution.edu/imsblis/lti13/token/<deployment_id>` |
| JWKS / Keyset URL | `https://sakai.your-institution.edu/imsblis/lti13/keyset` |

> **Note:** The Token URL includes a deployment ID that Sakai assigns when you register the tool. Copy the exact URL from the Sakai admin panel.
