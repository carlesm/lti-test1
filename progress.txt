- Delete is only allowed when `course.phase == Course.PHASE_SETUP`; return dashboard with `error` context var if not
- LTI role URIs come from `lti_tool.lti_core.constants.ContextRole` enum (e.g., `ContextRole.INSTRUCTOR.value`)
- Session keys for LTI data are defined in `projects/lti_views.py` as module-level constants (LTI_SUB_KEY, etc.)
- `lti_launch.ags_claim` returns the AGS endpoint dict; `lineitem` key holds the specific line item URL
- `lti_launch.context_claim` returns dict with `id` and `title` keys
- `lti_launch.roles_claim` returns a list of role URI strings
- The lti_required decorator checks for LTI_CONTEXT_ID_KEY in session to verify a valid LTI session
- URL namespacing: app_name="projects" in urls.py; use `projects:view_name` for reversal
